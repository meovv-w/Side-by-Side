# TongDao / TongDao-v2 审计结论

## 旧项目只读审计

`/home/lin/workspace/tongdao` 未做任何修改，也未迁移 `node_modules`。

可复用内容：

- `miniprogram/images/tabbar` 和 `images/markers`：体积小、无业务耦合，已选择性复制到 v2。
- `cloudfunctions/follow`、`privateMessage`、`poiChat`、`groupbuyJoin`、`orderPay`、`verifyCode` 等函数：可作为生产云函数拆分和字段设计参考。
- `pages/userProfile`、`settings`、`merchant`、`verify`：可作为页面命名和早期交互参考。
- 旧项目 `miniprogram-ci` 依赖和私钥：只用于本机微信 CI 编译，没有复制或提交到 v2。

不建议直接迁移：

- 旧页面和云函数之间存在较强字段/集合耦合，整批迁移会把历史数据模型和重复页面带入 v2。
- 旧项目依赖和历史内容较多，直接合并会扩大部署面并降低当前 V1 的可审计性。
- 旧后台逻辑不适合作为当前两个单文件交互后台的直接依赖。

## v2 原缺口与处理

| 原缺口 | 当前处理 |
|---|---|
| 4 Tab 只有基础列表，未按方案 UI 呈现 | 地图状态/图层/浮窗、消息分类、行程卡片、个人中心已重做 |
| 地图图层只是页面按钮和弹窗 | 改为地图标记开关、标记详情浮窗及导航/拨号/关注/转发操作 |
| 私信点击只显示提示 | 新增私聊页、关注关系、3 条限制和黑名单 |
| 行程加入没有队长审批 | 新增申请、待处理、通过/拒绝、退出和结束状态 |
| 发布行程字段不足 | 改为三步向导并补途经点、天数、里程、深度、计划和隐私 |
| 订单默认空、支付只叫“模拟支付” | 补完整商品、选券、支付结果、核销码、详情和退款状态 |
| 认证、邀请、客服、草稿是单按钮占位 | 补图片选择/活体步骤、奖励进度、FAQ/工单、草稿转正式行程 |
| “我的”缺社交和安全模块 | 新增主页、关注/粉丝、黑名单、勋章、紧急联系人、哨兵和隐私设置 |
| 两个后台只会弹 `Mock` 提示 | 重写为有登录、表单、表格、操作状态和浏览器持久化的后台 |
| README 与真实目录/能力不一致 | README、覆盖表、提交说明、测试清单和初始化 JSON 已同步 |

## 当前仍需外部条件

真实微信支付/退款/分账、高德实时数据、腾讯云 IM、OCR/活体、生产数据库、后台 RBAC、回调验签和正式审核发布不能只靠前端仓库完成。当前版本完成了这些能力的用户界面和业务状态呈现，正式上线仍需公司账号、密钥与部署环境。
