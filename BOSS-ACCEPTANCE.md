# 同路行老板验收测试清单

目标：证明方案中的 UI 和真实用户流程已经完整呈现。测试时不要只快速浏览页面，要完成操作并确认状态变化。

## 测试准备

1. 扫描 `preview-qrcode.png`，测试微信号必须是该小程序的管理员、开发者或体验成员。
2. 如果之前测试过，在地图右上角打开“图层”，点击“恢复初始体验数据”。
3. 用浏览器分别打开 `admin-merchant/index.html` 和 `admin-ops/index.html`，登录页已填入体验账号。

## 商家后台必测

1. 登录后确认经营概览和待办正常显示。
2. 发布商品，阶梯价填写 `1:48, 3:39.9, 6:34.9, 10:29.9`；保存后列表出现，可上下架。
3. 核销中心输入 `620318`；核销成功后订单状态和核销记录同时变化。
4. 修改店铺、银行卡、拉新券和救援服务，切换页面再返回，数据仍保留。
5. 查看结算、考核和消息通知，确认模块不是静态图片。

## 运营后台必测

1. 审核“小北”的车主认证，通过后状态变化。
2. 审核“同路补给铺”的商家资质，并调整已通过商家等级。
3. 在退款审核同意一条申请，再到全平台订单确认状态变为已退款。
4. 在结算管理触发待结算单，状态变为已结算。
5. 修改平台券预算、发放邀请奖励、修改同路值规则，刷新页面确认保留。
6. 归档/下架地点话题，处理客服工单和隐私投诉。

## 老板验收时要说明的边界

这版满足“方案 UI + 完整功能呈现”：用户可以从入口走到结果，数据有状态并持久化。以下不是测试失败，而是正式上线前必须由公司提供的外部条件：

- 微信支付、退款、分账商户权限。
- 高德地图实时 Key 和交通/天气/POI 服务。
- 腾讯云 IM 或其他实时消息服务。
- OCR/人脸活体、生产数据库、后台账号权限和审核发布。

演示视频建议依次录：地图图层与浮窗 → 行程申请/队长审批 → 群聊/私信 → 拼团用券/订单退款 → 商家核销 → 运营退款审核。控制在 6-8 分钟。
